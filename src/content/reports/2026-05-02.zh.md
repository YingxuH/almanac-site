# Almanac 每日预测 — 2026-05-02
## 软件工程的未来

### 执行摘要
昨天的焦点集中在企业强硬的 AI 绑定所带来的摩擦上，而今天则明显转向了透明、原生于工作流的 AI 集成以及对开源的倡导。开发者正积极抵制“黑盒”生成器，转而青睐能够直接接入现有复杂软件的工具，Adam AI CAD 套件的发布就是最好的证明。与此同时，安全领域依然严重分裂且情绪悲观，这源于一场双线危机：持续的跨国网络攻击导致 Ubuntu 的基础设施瘫痪，以及 AWS 在中东的物理数据中心遭到破坏。

### 今日核心信号

**1. 基础设施四面楚歌：Ubuntu 遭网络攻击与 AWS 物理受损**
一场协同的跨国网络攻击已导致 Ubuntu 的关键服务器基础设施离线超过一天，与此同时，有报道称 AWS 的物理数据中心因遭无人机严重打击受损，正在暂停对中东客户的计费。*重要性：* 未来 3 到 10 年的软件工程前景将越来越多地不仅由应用逻辑来定义，而是受制于基础设施的极端脆弱性。无处不在、高可用的云资源这一假设，正受到地缘政治实体战争和国家级网络破坏的双重根本性挑战。

**2. 抵制“黑盒”AI：Adam CAD 套件**
初创公司 Adam 发布了一款新的 AI 套件，该套件直接集成到 Autodesk Fusion 和 Onshape 中，明确避开了从提示词到 3D 模型的“黑盒”模式。相反，该 AI 能够以智能体的方式，在完全可见的情况下编辑用户现有的特征树。*重要性：* 这直接反映了今日 AI 编程领域积极情绪的飙升 (+0.30)。专业工程师拒绝采用掩盖底层架构的 AI。未来十年的企业级 AI 工具将严格要求对现有的应用状态进行透明、确定性的操作，而不是生成不透明的、最终定型的输出结果。

**3. 微软发布 Lib0xc 以提升系统编程安全性**
微软发布了 Lib0xc，这是一套紧邻 C 标准库的 API，专为鼓励更安全的系统编程实践而设计。*重要性：* 尽管 AI 代码生成取得了巨大进步，但业界认识到，底层系统逻辑（C、C++）仍然极其脆弱。此次发布表明，在未来 10-15 年内，保障遗留代码和底层代码的安全将需要结构化的 API 护栏，而不是仅仅依赖 AI 来编写内存安全的代码。

**4. 公共部门抵制封闭生态系统**
一个开发者联盟发起了一封公开信 (keepthingsopen.com)，要求英格兰国民保健署 (NHS England) 保持其代码库开源。*重要性：* 继昨天对微软和 Anthropic 封闭生态系统策略的强烈抵制之后，公共部门和医疗保健领域的开发者正在予以反击。专有且能力强大的企业级 AI 生态系统与透明、可审计的公共基础设施之间的紧张关系，很可能在未来 5 年内导致技术栈走向分化。

**5. "The Gay Jailbreak" 证明大模型漏洞仍未解决**
一个名为 "The Gay Jailbreak" 的新提示词注入框架（通过 ZetaLib）今天在 GitHub 上获得了广泛关注，展示了绕过大模型安全协议的新颖方法。*重要性：* 随着 AI 助手获得对本地文件系统和企业代码库更深层次的访问权限（如 Claude Code），复杂的提示词注入技术的泛滥构成了严重威胁。这必定会引发未来 3-5 年内专门的 AI 应用安全工具的大规模涌现。

**6. Claude Code 的插件生态系统通过 "Destiny" 得到扩展**
一名开发者发布了 *Destiny*，这是 Anthropic 的 Claude Code 的一个经典东亚占星术插件，它使用确定性的 Python 脚本来计算星盘，同时利用 Claude 生成叙述性的散文。*重要性：* 尽管 Anthropic 最近对提及竞争对手工具实施了强硬的限制，但社区仍在热情地为 Claude Code 构建可扩展的本地插件。这标志着人们希望 AI 助手能充当可扩展的终端操作系统，而不是僵化的企业聊天界面。

### 预测相关性

**15. “到 2030 年，AI 编程工具的最大赢家将是非科技公司（银行、制造商、零售商），而不是科技公司”**
Adam AI CAD 套件的发布（直接集成到 Autodesk Fusion 等工具中）加速了这一趋势，将自主的特征树操作引入了传统的机械工程和制造工作流。

**28. “针对 AI 编程智能体的提示词注入攻击将在 2028 年成为新的 OWASP Top 10 类别”**
GitHub 上 "Gay Jailbreak" 技术的公布突显了新型提示词注入漏洞的快速演变和记录，这些漏洞正日益威胁着集成的 AI 编程智能体。

**36. “到 2030 年，系统编程技能（Rust、C++、内核开发）相对于 Web 开发的薪资溢价将高于 2025 年”**
微软开发并发布 Lib0xc 凸显了底层 C 编程持久的复杂性和关键的安全要求，进一步巩固了专业系统工程技能的价值。

### 未来趋势差异
*与昨天相比具体发生了什么变化？*
昨天的叙事主基调是“认知债务”（Epistemological Debt），以及对开发者因不透明的 AI 系统而脱离基础工程的担忧。今天，钟摆摆了回来。Adam CAD 套件的广受好评以及微软 Lib0xc 的推出证明，市场正在积极纠偏。工程师们正在构建要求透明度（可见地编辑特征树）并强制执行基础安全（系统 API）的工具，而不是将它们抽象掉。
